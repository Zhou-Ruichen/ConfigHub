# Web UI Quality Guidelines

## Accessibility

- Use semantic HTML.
- Forms and buttons need labels.
- Tables need clear headers.
- Copyable commands should also be selectable text.

## Product Quality

- The UI must make target paths and safety policy visible before users sync/apply.
- Avoid hidden destructive actions.
- Avoid SPA-only flows for the MVP.
- Keep pages useful on narrow screens, but prioritize operational clarity over marketing layout.

## Validation

- Manually open the served UI.
- Confirm profile/template/bundle pages render without JavaScript.
- Confirm bootstrap commands match CLI syntax.
